# Changelog: Deploy Fevereiro 2025

## checkout v2.15.0 (2025-02-10)
- **Feature**: Novo sistema de cupons com validação server-side
- **Fix**: Null check no cálculo de frete (ref: postmortem 2024-12)
- **Perf**: Cache de consulta de estoque (Redis, TTL 30s)
- **Deps**: Atualização Spring Boot 3.2 → 3.3

## payments v1.8.2 (2025-02-12)
- **Fix**: Timeout de 30s → 10s para gateway de pagamento (reduz hanging requests)
- **Feature**: Suporte a PIX parcelado
- **Observability**: Adicionadas métricas de latência por gateway

## inventory v3.1.0 (2025-02-14)
- **Feature**: Reserva temporária de estoque (lock de 15 min no checkout)
- **Fix**: Race condition na atualização de quantidade (pessimistic lock)
- **Infra**: Migração para connection pool HikariCP (max 50 conexões)

## api-gateway v1.12.0 (2025-02-15)
- **Feature**: Rate limiting por tenant (100 req/s default)
- **Security**: Validação de JWT com rotação de chaves
- **Perf**: Compressão gzip para responses > 1KB

## Notas de Operação
- Monitorar memory do checkout após cache de estoque (estimado +200MB RSS)
- Payments: verificar se timeout de 10s não causa aumento de erros com gateway lento
- Inventory: lock de 15 min pode causar contenção em pico — monitorar métricas de lock wait
